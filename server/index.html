Testing REST API using Postman

Registering User

To Register a user select POST from the dropdown menu use localhost:/api/register?apiKey= as the address Select the body field and select x-www-form-urlencoded Add the fields name, email and password and enter a value for all of them

Logging in User

Logging is similar to registering a user but your url is localhost:/api/login?apiKey= and you only need email and password fields you need the token from the JSON response

Getting User Profile

To get the the user profile select GET and use localhost:/api/register?apiKey=, Select headers and enter Authorization as key and the JWT token as the value. The JSON response will have user information

Adding a Note

To add a Note select POST and use localhost:/api/addNote?apiKey=, like getting the user profile you will need to set the token in the header. Go to body and use the key note and for the value type in a note (text)

Removing a Note

To remove a Note select POST and use localhost:/api/removeNote?apiKey=, you will also need to pass the token in the header and for the body id is key and for the value enter the _id of the note given by the profile request.

Requests

Url Type Params Description Response
localhost:3000/api/register?apiKey=<> POST name - Name of user,
email - email of user,
password - password of user		 Registers the user with the application JSON response on whether successful or not
localhost:3000/api/login?apiKey=<> POST email - email of user,
password - password of user		 Authenticates the user and returns Javascript Web Token JSON response with Javascript web token
localhost:3000/api/profile?apiKey=<> POST Need to pass the JWT token in Authorization header Gets profile information about user JSON response with user profile
localhost:3000/api/addNote?apiKey=<> POST Need to pass the JWT token in Authorization header
note - contents of the note		 Adds the note to the user JSON response whether successful or not
localhost:3000/api/removeNote?apiKey=<> POST Need to pass the JWT token in Authorization header
id - id of the note		 Removes the note from the user JSON response whether successful or not
